🔹 데이터 전처리 작업 절차

데이터 불러오기

국토교통부 실거래가 공개시스템에서 수집한 아파트 매매 실거래가 CSV를 불러왔습니다.

파일 상단에 포함된 공지 문구는 skiprows 옵션으로 제거했습니다.

불필요한 컬럼 제거

분석과 예측에 불필요한 NO, 번지, 본번, 부번, 매수자, 매도자, 도로명, 해제사유발생일, 등기일자, 동 컬럼을 제거했습니다.

숫자형 변환

거래금액(만원)에서 쉼표(,)와 특수기호를 제거하고 정수형으로 변환했습니다.

계약년월, 계약일, 층, 건축년도는 문자형 데이터가 섞여 있어 to_numeric을 통해 숫자형으로 변환하고 결측치는 0으로 대체했습니다.

범주형 데이터 인코딩

시군구, 단지명, 거래유형, 중개사소재지와 같은 범주형 변수를 원-핫 인코딩(one-hot encoding)하여 모델이 활용할 수 있도록 변환했습니다.

학습/테스트 분리

최종적으로 전처리된 데이터를 train_test_split을 이용해 학습:테스트 = 8:2 비율로 분리했습니다.

🔹 알고리즘 3개 선정 이유

Linear Regression (선형 회귀)

가장 기본적인 회귀 알고리즘으로, 입력 변수와 목표 변수 간 선형 관계를 가정합니다.

단순 baseline 성능 확인용으로 선정했습니다.

Decision Tree Regressor (의사결정나무)

비선형 관계를 잘 설명할 수 있고, 데이터의 분포를 트리 구조로 나누어 직관적으로 예측합니다.

전용면적, 층, 건축년도, 지역 등 아파트 가격에 영향을 주는 요인이 비선형적이라 적합하다고 판단했습니다.

Random Forest Regressor (랜덤포레스트)

여러 개의 결정트리를 앙상블해 과적합을 줄이고 예측 성능을 강화하는 모델입니다.

실제 가격 예측 문제에서 높은 성능을 보이는 대표적인 회귀 알고리즘이므로 선정했습니다.

🔹 성능 지표 선정 이유

RMSE (Root Mean Squared Error): 예측 오차를 제곱해 평균한 뒤 제곱근을 취한 값으로, 큰 오차에 민감하게 반응하여 모델의 안정성을 평가할 수 있습니다.

MAE (Mean Absolute Error): 절댓값 기준의 평균 오차로, 실제 거래금액과 예측값의 평균 차이를 직관적으로 보여줍니다.

R² (결정계수): 모델이 실제 데이터를 얼마나 잘 설명하는지(설명력)를 나타내며, 1에 가까울수록 성능이 우수함을 의미합니다.

🔹 성능 결과 및 해석

Linear Regression

RMSE와 MAE가 비정상적으로 크고, R²가 음수로 나타났습니다.

이는 데이터가 비선형적이고 범주형 변수가 많아 단순 선형 모델로는 적합하지 않음을 의미합니다.

Decision Tree

RMSE ≈ 5천, MAE ≈ 3천, R² ≈ 0.94로 선형 회귀보다 훨씬 좋은 성능을 보였습니다.

개별 특성에 따라 가격을 잘 분리해 설명력을 확보했습니다.

Random Forest

RMSE ≈ 4천, MAE ≈ 2.5천, R² ≈ 0.96으로 세 모델 중 가장 우수했습니다.

여러 트리를 앙상블하여 안정성과 예측 정확도를 동시에 확보했습니다.

-> 결론적으로, 아파트 실거래가 예측에는 Random Forest Regressor가 가장 적합한 모델임을 확인할 수 있었습니다.
